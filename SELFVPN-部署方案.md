## Problem Statement

需要一个简单可靠的自建代理方案，让自己和家人朋友（非技术用户）都能便捷地翻墙使用。从零开始，没有现有服务器。客户端使用 Clash Verge，需要支持分流规则。

## What Makes This Cool

约 1 小时搭建一个全家可用的翻墙服务。VLESS + REALITY 协议伪装成正常 HTTPS 流量，GFW 几乎无法识别。家人只需在 Clash Verge 中导入一个订阅链接就能用，国内网站自动直连，国外网站自动走代理。

## Constraints

- 用户包括不懂技术的家人朋友，配置必须极简
- 从零开始，需要购买 VPS
- 客户端使用 Clash Verge Rev（Windows/macOS）、Clash Meta for Android（Android）、Stash 或 Shadowrocket（iOS，付费）
- 追求简单可靠 + 抗封锁
- 预算敏感，个人使用

## Premises

1. 核心需求是翻墙访问国外网站
2. 自建优于商业 VPN/机场（可控性、隐私、稳定性、不会突然跑路）
3. VLESS + REALITY 是当前抗 GFW 封锁能力最强的协议组合
4. Clash Verge 是客户端首选（分流规则、UI 友好）
5. 单节点方案对个人使用场景足够
6. 需要一个订阅链接或配置文件分发给家人

## Approaches Considered

### Approach A: Xray + VLESS + REALITY（手动部署） -- CHOSEN

- **Summary**: 在 VPS 上安装 Xray，配置 VLESS + REALITY 协议，生成 Clash 配置文件分发给家人
- **Effort**: S-M（约 1-1.5 小时，含购买 VPS）
- **Risk**: Low
- **Pros**: 抗封锁最强；Clash Verge 原生支持；分流规则开箱即用；性能优秀
- **Cons**: 比 wg-easy 步骤稍多；没有 Web 管理面板；新增用户需手动编辑配置

### Approach B: 3X-UI 面板（Web UI 管理）

- **Summary**: 部署 3X-UI Web 面板，图形化管理 Xray 节点和用户
- **Effort**: S（半小时）
- **Risk**: Low-Medium
- **Pros**: Web UI 管理节点和用户；一键生成订阅链接；可视化流量统计
- **Cons**: 面板本身可能有安全风险；多一层依赖；面板特征可能被识别

### Approach C: Docker + sing-box

- **Summary**: 用 Docker 部署 sing-box（Clash Meta 内核的服务端版本）
- **Effort**: M（1-2小时）
- **Risk**: Medium
- **Pros**: 现代架构；配置灵活；Docker 环境一致
- **Cons**: 文档较少；社区不如 Xray 成熟

## Recommended Approach

**方案 A: Xray + VLESS + REALITY 手动部署**

理由：最稳定、最成熟、社区资料最多。REALITY 协议不需要域名和证书，利用别人的网站做伪装，部署门槛最低。虽然没有 Web 面板，但个人使用不需要频繁加用户，手动配置一次就完事。

### 第二步：连接服务器

```bash
# Windows Terminal / PowerShell 直接用
ssh root@你的服务器IP
# 输入密码（输入时不显示，正常现象）
```

### 第三步：系统初始化

```bash
# 更新系统
apt update && apt upgrade -y

# 安装必要工具
apt install -y curl wget unzip

# 开启 BBR 加速（提升网络性能）
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p
```

### 第四步：安装 Xray

```bash
# 官方一键安装脚本
bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install
```

安装完成后 Xray 位于 `/usr/local/bin/xray`，配置文件在 `/usr/local/etc/xray/config.json`。

### 第五步：生成密钥

```bash
# 生成 X25519 密钥对（REALITY 需要）
xray x25519
```

输出类似：
```
Private key: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Public key:  YYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYY
```

**记下这两个值！** Private key 放服务端配置，Public key 放客户端配置。

```bash
# 生成 UUID（用户标识）
xray uuid
```

输出类似：`a1b2c3d4-e5f6-7890-abcd-ef1234567890`

```bash
# 生成 shortId（可选，用于增强安全性）
openssl rand -hex 8
```

输出类似：`6ba85179e30d4fc2`

**把 UUID、Private key、Public key、shortId 全部记下来！建议保存到密码管理器。**

### 第六步：配置 Xray 服务端

```bash
cat > /usr/local/etc/xray/config.json << 'XRAYEOF'
{
  "log": {
    "loglevel": "warning"
  },
  "inbounds": [
    {
      "listen": "0.0.0.0",
      "port": 443,
      "protocol": "vless",
      "settings": {
        "clients": [
          {
            "id": "你的UUID",
            "flow": "xtls-rprx-vision"
          }
        ],
        "decryption": "none"
      },
      "streamSettings": {
        "network": "tcp",
        "security": "reality",
        "realitySettings": {
          "show": false,
          "dest": "www.microsoft.com:443",
          "xver": 0,
          "serverNames": [
            "www.microsoft.com"
          ],
          "privateKey": "你的Private_Key",
          "shortIds": [
            "",
            "6ba85179e30d4fc2"
          ]
        }
      },
      "sniffing": {
        "enabled": true,
        "destOverride": [
          "http",
          "tls",
          "quic"
        ]
      }
    }
  ],
  "outbounds": [
    {
      "protocol": "freedom",
      "tag": "direct"
    },
    {
      "protocol": "blackhole",
      "tag": "block"
    }
  ]
}
XRAYEOF
```

**替换三个值**：
- `你的UUID` → 第五步生成的 UUID
- `你的Private_Key` → 第五步生成的 Private key
- `shortIds` 中的 `6ba85179e30d4fc2` → 可以自定义，也可以保留

**关于伪装目标**（dest 和 serverNames）：
- `www.microsoft.com` 是伪装目标，GFW 看到的是你在访问微软网站
- 可以换成其他大网站：`www.apple.com`、`www.amazon.com`、`gateway.icloud.com`
- 要求：目标网站支持 TLSv1.3 和 H2
- 验证方法：`curl -I --http2 -sS https://目标网站 | head -5`，看到 `HTTP/2 200` 就可以

### 第七步：启动 Xray

```bash
# 启动服务
systemctl restart xray

# 设置开机自启
systemctl enable xray

# 检查状态
systemctl status xray

# 如果有问题，查看日志
journalctl -u xray -f
```

### 第八步：配置防火墙

**!! 警告：务必先放行 SSH 端口，否则 `ufw enable` 后你将被锁在服务器外面 !!**

```bash
# 先放行 SSH
ufw allow 22/tcp

# 放行 Xray 端口
ufw allow 443/tcp

# 启用防火墙
ufw enable
```

**注意**：如果你使用 Oracle Cloud、AWS 等云厂商，除了 ufw 之外，还需要在云控制台的**安全组/Security List** 中放行对应端口。ufw 只管服务器内部防火墙，云厂商有额外一层。
```

### 第九步：生成 Clash Verge 配置文件

在本地电脑创建一个文件 `selfvpn.yaml`：

```yaml
mixed-port: 7890
allow-lan: false
mode: rule
log-level: info

dns:
  enable: true
  enhanced-mode: fake-ip
  fake-ip-range: 198.18.0.1/16
  nameserver:
    - https://dns.alidns.com/dns-query
    - https://doh.pub/dns-query
  fallback:
    - https://1.1.1.1/dns-query
    - https://8.8.8.8/dns-query
  fallback-filter:
    geoip: true
    geoip-code: CN

proxies:
  - name: "SELFVPN"
    type: vless
    server: 你的服务器IP
    port: 443
    uuid: 你的UUID
    network: tcp
    tls: true
    udp: true
    flow: xtls-rprx-vision
    servername: www.microsoft.com
    reality-opts:
      public-key: 你的Public_Key
      short-id: "6ba85179e30d4fc2"
    client-fingerprint: chrome

proxy-groups:
  - name: "代理"
    type: select
    proxies:
      - SELFVPN
      - DIRECT

  - name: "国外网站"
    type: select
    proxies:
      - 代理
      - DIRECT

  - name: "国内网站"
    type: select
    proxies:
      - DIRECT
      - 代理

rules:
  # 国内域名直连
  - GEOSITE,cn,国内网站
  - GEOIP,CN,国内网站

  # 常用国外网站走代理
  - GEOSITE,google,国外网站
  - GEOSITE,youtube,国外网站
  - GEOSITE,twitter,国外网站
  - GEOSITE,facebook,国外网站
  - GEOSITE,telegram,国外网站
  - GEOSITE,github,国外网站
  - GEOSITE,openai,国外网站

  # 其他默认走代理
  - MATCH,代理
```

**替换三个值**：
- `你的服务器IP` → VPS 的 IP 地址
- `你的UUID` → 第五步生成的 UUID
- `你的Public_Key` → 第五步生成的 Public key（注意是 Public，不是 Private）

### 第十步：导入 Clash Verge

**你自己的电脑**：
1. 下载安装 Clash Verge Rev：https://github.com/clash-verge-rev/clash-verge-rev/releases
2. 打开 Clash Verge
3. 左侧菜单 → "配置"（Profiles）
4. 点击 "新建"（New）→ "本地文件"（Local）
5. 选择刚才创建的 `selfvpn.yaml`
6. 点击启用该配置
7. 主界面开启系统代理

**给家人**：
1. 把 `selfvpn.yaml` 文件发给他们（**注意**：微信可能压缩或拦截 .yaml 文件，建议压缩成 .zip 后发送，或用邮件附件、AirDrop、网盘链接传输）
2. 让他们装 Clash Verge（Windows/Mac）或 Clash Meta for Android / Stash（iOS）
3. 导入配置文件
4. 开启即可

**手机客户端**：
- **Android**: Clash Meta for Android（从 GitHub Releases 下载 APK）
- **iOS**: Stash（App Store，付费）或 Shadowrocket（App Store，付费）

### 第十一步：验证连接

1. 开启 Clash Verge 系统代理
2. 浏览器访问 `https://www.google.com`，能打开就成功
3. 访问 `https://whatismyipaddress.com/`，确认 IP 显示为服务器 IP
4. 访问 `https://www.baidu.com`，确认直连（不走代理）

## 多用户管理

如果需要给不同家人创建独立账号（可以单独撤销权限）：

在服务端配置的 `clients` 数组中添加更多 UUID：

```json
"clients": [
  { "id": "uuid-自己用", "flow": "xtls-rprx-vision" },
  { "id": "uuid-爸爸用", "flow": "xtls-rprx-vision" },
  { "id": "uuid-妈妈用", "flow": "xtls-rprx-vision" }
]
```

每个人的 Clash 配置文件中使用各自的 UUID。

添加后重启 Xray：`systemctl restart xray`

## 故障恢复

### IP 被封

1. 在 VPS 控制台更换 IP（Vultr 支持销毁重建，按小时计费损失极小）
2. SSH 登录新服务器，重新执行第三到第七步
3. 更新所有 Clash 配置文件中的服务器 IP
4. 重新分发给家人

### Xray 服务异常

```bash
# 查看状态
systemctl status xray

# 查看日志
journalctl -u xray --no-pager -n 50

# 重启
systemctl restart xray
```

### 连接突然断开

排查顺序：
1. 服务器能 SSH 吗？→ 不能则 VPS 宕机或 IP 被封
2. Xray 在运行吗？→ `systemctl status xray`
3. 端口通吗？→ 本地运行 `telnet 你的服务器IP 443` 看端口是否可达
4. 换个伪装目标试试 → 修改 `dest` 和 `serverNames`

## 安全加固（建议首次运行后尽快配置）

1. **修改 SSH 端口**：
   ```bash
   sed -i 's/#Port 22/Port 你的端口/' /etc/ssh/sshd_config
   systemctl restart sshd
   ufw allow 你的端口/tcp
   ```

2. **禁用密码登录，改用 SSH Key**：
   ```bash
   # 本地生成 key（如果没有的话）
   ssh-keygen -t ed25519
   # 复制到服务器
   ssh-copy-id -p 你的端口 root@你的服务器IP
   # 禁用密码登录
   sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
   systemctl restart sshd
   ```

3. **设置自动更新**：
   ```bash
   apt install -y unattended-upgrades
   dpkg-reconfigure -plow unattended-upgrades
   ```
